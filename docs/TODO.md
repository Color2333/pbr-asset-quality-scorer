## 一、VLM 方案（重点）

### A1. Qwen 特征提取 + 输出头（路线 A）⭐ 新方向，干净对照
把 Qwen-VL 当**特征提取器**（取 hidden states），接自定义头（MLP / EMD / ordinal），区别于现在的"生成式读 6 digit token"。
- **入手点**：
  1. 复用 `scripts/precompute_vlm_prior.py`（已能抽 answer 位 hidden 3584d）→ 全量抽 hidden 缓存
  2. 接 EMD/ordinal 头（直接复用 `train_multitask.py` 的 `_emd_loss`）
  3. 冻结 Qwen 只训头（快、稳），对比"生成式读出"哪个好
- **现状**：没做过。当前 VLM 用的是原生词表头读 6 数字。
- **为什么值得**：头容量可控、可上 EMD/extremes/序数 loss、可加风格条件 FiLM；是"VLM 特征 vs VLM 生成"的干净 A/B。
- **成本**：抽 hidden 一次（GPU 几小时）+ 训头（分钟级，CPU/GPU 都行）。

### A2. 全通道输入 + Prompt 修改（注入现实知识）⭐ 部分新
- **入手点**：
  1. **把 `CH_PROMPT` 升级成 SOP 分档准则**（每通道把 0–5 各档判据写进 prompt，注入"非金属空map=对、风格化允许偏离"等现实知识）
  2. **先零样本测**（无 train/test prompt 不一致问题）→ 看 SOP-prompt 比简短 prompt 涨多少
  3. 有效再"带 SOP-prompt 重训"
- **现状**：multi-channel 全通道输入**已试→对 B 组盲区无效**；但 **SOP 分档 prompt 没系统做过**（当前 prompt 是简短版）。
- **预期**：微调侧大概率边际（SFT 已吸准则，"prompt 边际已尽"有据）；**零样本/预标注工具侧有效**——这才是它的主场。
- **成本**：起草 prompt（零）+ 零样本 eval（GPU 短）。

### A3. RLHF / 对齐人类偏好 ⭐ 新方向，可能特别契合"两端"
- **入手点**：
  1. RLHF/DPO 需**偏好对**。先从现有 0–5 标签**构造 pairwise**（分差≥2 的配成 A≻B），用 **DPO**（比 PPO 简单稳）做小实验
  2. 评估仍看守护分/两端 AUC
- **现状**：没做。当前是 SFT 拟合点估计。
- **关键洞察**：偏好学习是**排序/比较**，不在乎绝对刻度——**正好契合我们"两端才是重点、绝对分有噪声"的发现**；pairwise 对噪声比绝对分更鲁棒（差一档的噪声在排序里影响小）。
- **风险**：偏好对仍从噪声标签构造，受标签限；但鲁棒性更高，值得一试。真人偏好标注更强但贵（连回多人重标）。
- **成本**：中（DPO 训练一版）。

### A4. LoRA 视觉编码器 ⚠️ 已试，单独做无效
- **现状**：**vision-LoRA 全量（full60k, vision_lora=True）落在同一 ~0.79 天花板，没突破**；DINOv2 全解冻净有害。DB-3DME 说"适配视觉编码器是关键"对我们不成立（瓶颈是标签非编码）。
- **入手点（若再做）**：只值得**和 A2/增亮组合**——放大输入信号 + 轻量 vision-LoRA 让编码器学会编码它；必须参数高效以免过拟合噪声。
- **优先级**：**低**（除非增亮实验先证明"信号可见"有效）。

### A5. 尺寸/分辨率确认（小实验）
- **入手点**：固定一个 VLM checkpoint，**eval 时扫 `--max-pixels`**（100352 / 147456 / 200704），看 metallic 及细节通道变化。⚠️ 严谨结论需 train/test 同分辨率 → 真要定论得同分辨率重训一小版。
- **现状**：DINOv2/ConvNeXt 224 vs 448 **已试→±0.01 噪声内，metallic 略降**。VLM 的 max-pixels 没干净隔离过。
- **预期**：大概率也是 ±0.01（感知瓶颈非分辨率）。**低成本确认、排除即可**，别投太多。

## 二、DINOv2 方案

### B1. 通道专用表示（roughness / normal 等）⭐ 有针对性
每通道按物理意义做**专用输入变换**，打对应短板：
- **metallic**：对比拉伸增亮（**进行中**，暴露近黑低幅判别灰）
- **roughness**：标量场 → 值域拉伸 / 直方图 / per-material 统计（SOP："值域 0–1 有层次"是 5 分关键）
- **normal**：RGB 编码 → 解码成角度/曲率，或显式检测**红绿反转 / 纯蓝占比**（对应 Yes/No "法线反转"标记）
- **base_color**：检测**假 AO / 发光溢色**（对应 Yes/No 标记）
- **入手点**：先在 `train_multitask.py` 的 `__getitem__` 里加各通道的输入变换开关（metallic_stretch 已是范例），逐个小实验
- **现状**：metallic 增亮跑着；normal/roughness/base 专用表示没做。
- **预期**：normal 反转 / base 假AO 是**良定义缺陷**，专用表示 + 二元头可能真涨（不像 0–5 工艺分被噪声封顶）。
- **成本**：每个一个小实验（DINOv2 冻结骨干，快）。

- **基线层级**（已建一部分）：平凡图像统计 ≈0 ｜ **CLIP 线性探针 0.27–0.80** ｜ 深度模型 0.63–0.91 ｜ **人类一致性上界=待补**
