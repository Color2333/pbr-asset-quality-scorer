# PBR 材质质量评估器 — 技术报告

> 截至 2026-06-04，覆盖 `outputs/runs/` 全部实验（含 ConvNeXt/DINOv2 单通道、多任务、EMD/序数头、正则栈、synthneg 家族）。
> 本报告以**分析与论证**为主、指标为辅：每个结论都尽量回答"为什么会这样"，并交代是用什么实验**证明**的，而不是只罗列数字。

---

## 0. 一句话结论（最终）

我们训练了一个**对齐人类偏好的 PBR 材质质量评估器**：给定渲染图 + 各通道贴图，对 4 个通道（base_color / normal_map / roughness / metallic）各预测 0–5 质量分，北极星指标为 **SRCC**（与人类打分的排序相关）。

**生产模型 = DINOv2 多任务 + EMD 分布头**，test 集表现：

| 通道 | roughness | base_color | normal_map | metallic | **均值** |
|---|:---:|:---:|:---:|:---:|:---:|
| test SRCC | **0.896** | 0.842 | 0.807 | 0.624 | **0.792** |

**核心结论**：三个通道（roughness/base_color/normal）已逼近其单标注者一致性天花板；**metallic 长期卡在 0.62–0.63，且经穷尽性实验证明这是硬天花板而非模型能力问题**。我们沿三条战线全部打到尽头——**架构**（容量↑/跨通道/序数/EMD/FiLM）、**训练侧**（正则栈/drop_path/辅助监督/LLRD/解冻策略）、**信息注入**（合成漏标负样本 ×3 变体）——metallic test 在所有干预下都纹丝不动。其中**合成漏标负样本**是关键的反事实判别实验：我们人为制造"标准答案确定无误"的漏标样本喂给模型，若 metallic 能涨则说明信息可恢复、还有空间，结果**完全不涨、漏标检测召回反而下降**，从正面证明了天花板由**不可消除的标注歧义 + 单资产输入里"该不该是金属"的信息不足**共同设定。

真正稳定有效的提升只来自两个通用杠杆——**更强主干（DINOv2）** 和 **多任务共享表示**，各 ~+0.02–0.04 SRCC。EMD 分布头在不降 SRCC 的前提下额外拿到分类级精度（exact acc +0.02，且"差一档轻罚、差多档重罚"对齐"5→4 可接受"的产品容忍度）。

**产品级重定位（§8，本阶段最重要的结论）**：纠结"孤立 metallic SRCC"其实定错了北极星——SOP 的真实目的是**分层筛选训练数据**。在产品目标上我们其实已经**赢了**：FinalScore SRCC **0.831**、Tier 5 档 within-1 准确率 **98.6%**、"筛 Tier1-2 高质量"二分 AUC **0.924**。而且实测 FinalScore = 5 个维度分的加权和（R²=0.998），其中 **metallic 权重最低（0.15）、normal 最高（0.35）**——我们死磕的 metallic 恰是产品里最不重要的通道。本阶段还落地了**两个真正的正向交付物**：①尾部过采样把 normal 满分召回从 10% 救到 62%（零 SRCC 代价，直接服务高质量筛选）；②缺陷检测器（文字 AUC 0.93 / 法线异色 0.89，可作审核 flag）。

## 1. 任务定义与评估口径

**目标**：模型应复现人类审核员对材质质量的偏好排序。因此**北极星指标是 SRCC**（Spearman 排序相关），而非 MAE，更非二分类 F1——评估器的价值在于"排得和人一样准"。

**评估口径**：所有数字若无特别说明，均为 **test split（4917 样本）** 上的 SRCC。早期一些结论混用过 val，而 val→test 普遍掉 0.02–0.04（泛化 gap），下文统一用 test。

**一个被文献校准的关键认识（决定一切的标尺）**：
NR-IQA 文献中 SRCC 是衡量"预测分 vs 人类 MOS"的标准指标，但它有上限——**人类标注者之间的一致性就是模型的天花板**：

| 参照 | SRCC | 说明 |
|------|:---:|------|
| 合成失真 benchmark（LIVE/TID2013/KADID）SOTA | 0.93–0.98 | 简单人工失真，不可比 |
| 真实场景 benchmark（KonIQ-10k）SOTA | 0.92–0.93 | 但其 MOS 有 ~9–57 票/图 |
| 高质量/难任务（UHD-IQA）最好方法 | **0.77** | 与我们难度相近 |
| 纹理 IQA（FLT texture DB）最好方法 | **0.82**（作者自评"仍偏低"） | **纹理内容本身就难** |
| 最相近工作 BRDF-NQM（2025，材质质量） | **0.67** | **和我们 metallic 同量级** |
| KonIQ 人类两半组一致性（57 票/图） | 0.973 | "票数充足"时的人类天花板 |

**我们的数据是单标注者**（连"几票/图"都没有），标签噪声只会更大。结论：**对单标注者的真实材质质量任务，0.8 量级的 SRCC 已是合理的好成绩，不存在"轻松涨到 0.9+"的空间。**

## 2. 数据

**规模**：49165 个 3D 资产，train/val/test = 39329 / 4918 / 4918（≈ 8:1:1，按 pbrType 分层采样）。

**渠道来源**：抓取自 16 个来源，高度长尾，头部 5 个占 95%：

| 渠道 | 数量 | 占比 | 备注 |
|------|---:|---:|------|
| **sketchfab** | 29029 | **59.0%** | 绝对主力，社区 3D 模型站 |
| **3d66** | 6871 | **14.0%** | 国内室内/家具模型站 |
| sketchfab-objaverse | 4405 | 9.0% | Objaverse 子集 |
| games | 3724 | 7.6% | 游戏拆包资产 |
| unreal | 2682 | 5.5% | UE 商城 |
| abo / pbrmax / kitbash / megascan / hssd / project-aria | ~2400 | ~4.9% | 专业 PBR / 扫描 / 合成 |
| vroidhub / daz / evermotion / … | <50 | <0.1% | 长尾，可忽略 |

> 60% 来自 sketchfab（质量参差的社区上传），决定了数据整体偏"野生"——呼应 §1"我们的任务更接近 in-the-wild IQA（SOTA ~0.88）而非合成失真（0.95+）"。

**材质类型（pbrType）**：physical 88.0% / stylized 8.4% / uncertain 3.6%。3d66 几乎全 physical（93%），games 风格化最多（stylized 14%），unreal 的 uncertain 高达 21%（标注置信度低，是噪声来源之一）。SOP 明确 physical 与 stylized 判断规则不同。

**质量分层（tier）**：Tier3 游戏级 32.2% / Tier4 手游级 33.7% / Tier5 勉强可用 23.5% / Tier2 高级AAA 10.3% / **Tier1 电影级 0.4%**。**高端资产极度稀缺**——从数据层面解释了所有通道"高分段难学"。

**整体分（finalScore，0.5–5）**：mean 2.30 / median 2.30，钟形集中——[1,2) 占 34%、[2,3) 占 37%，[4,5) 仅 2.9%、[0,1) 仅 5.3%。**两端都稀少，优质样本尤其不足。**

**通道与渲染**：每资产 7 个通道图（render / base_color / roughness / metallic / normal_map / white_model / white_with_normal），**同机位、同几何、像素对齐**，源 2048²，预缓存为 uint8 memmap（224 与 448 两套；CLIP CLS 离线预计算 render+base_color 各 768-d）。

**标注**：每资产由**单个标注者**对 4 通道各打 0–5 分 + finalScore + 缺陷标记（文字/伪AO/法线异色/法线翻转）。**单标注、无多人交叉**——这是"无法直接测人类一致性天花板"的根源。

## 2b. 方法

### 2b.1 输入与特征
- **图像分支**：单通道贴图 [3,224,224]（或 448）。增广 = 随机翻转 + （可选）RandomErasing cutout；ImageNet 归一化。**注意 normal 不做 color jitter**（会破坏法线方向）。
- **语义分支（CLIP）**：`ViT-L-14/openai`，对 render 与 base_color 各取 CLS（768-d）拼成 **1536-d**，离线预计算存盘。拼接顺序 `[base_color(768), render(768)]`。

### 2b.2 主力架构
两代主干，接口一致（经注册表 `build_model` 派发）：
- **ConvNeXt-Base（90M）**：3 尺度 AttentionPool2d → CrossModalFusion(Q=3尺度, KV=2个CLIP CLS) + clip_direct 旁路 → 融合 MLP → 多头。
- **DINOv2 ViT-L/14 reg4（308M，主力）**：取第 8/16/24 层 patch token 池化作伪多尺度，dynamic_img_size 支持变分辨率，复用同一套 CrossModalFusion。**DINOv2 dense 自监督特征对材质纹理更友好，全通道持平或微涨。**

### 2b.3 多任务共享主干（本项目最有效的单一改动）
一个主干被 4 通道联合训练；4 个独立 score/binary 头；per-channel 损失权重（metallic 1.5×）；metallic 梯度 ×0.5 回流主干（防噪声标签污染共享表示）。**机制**：强通道的高质量梯度逼主干学到更好的通用材质表示，弱通道 metallic"白嫖"——真正的"强通道带弱通道"。推理时各通道独立。

### 2b.4 输出头（三种，可按通道配置）
| 头 | 形式 | 损失 | 取分 |
|---|---|---|---|
| **回归**（默认） | sigmoid×5 | Huber(δ=1.0) | 直接 |
| **序数 CORAL** | 5 个累积 logit | CoralRankLoss | 期望分 |
| **EMD/NIMA（生产用）** | 6-bin 概率分布 | squared EMD（累积分布差） | Σ(p·k) |

- **复合损失**：主损 + 0.2·BCE(valid/invalid) + 0.1·BCE(缺陷) + 0.05·成对 margin 排序（直接优化排序保真，与 SRCC 同向）。
- **EMD 为何选作生产头**：小实验+全量均证实，EMD 既保留序数性（SRCC 不输回归 0.792 vs 0.785）又用分布建模拿到分类级 acc（exact acc 0.589→0.607），**且天然"差一档轻罚、差多档重罚"**（5→4 损失 1.0，5→0 损失 5.0）——是产品容忍度"5 分误判成 4 分可接受、其他要进步"的原则性实现。序数头则是 SRCC↑但 acc↓的帕累托权衡，不如 EMD。

### 2b.5 正则与训练策略
- **优化器**：AdamW，lr 1e-4，wd 1e-4；CosineAnnealingLR。AMP bf16。
- **渐进解冻**：ep0–4 头部 → ep5 解 stage4 → ep10 解 stage3+4 → ep15 全解冻。**⚠️ 见 §5.3：全解冻在本数据规模下有害，已改用 LLRD + 只浅解冻 stage4。**
- **LLRD**（layer-wise LR decay 0.65）：backbone 低层 lr 指数衰减（block0 ~5e-9、顶层 1e-4），治全解冻过拟合。
- **辅助监督**（正则用）：从 base_color 表示额外预测 tier/pbrType/4 个缺陷标记（现成标签，从未用过），逼共享主干学通用材质理解而非死记噪声分。
- **drop_path 0.2**（stochastic depth）+ **RandomErasing**：抗过拟合正则。
- **checkpoint**：按 val srcc_mean 存 `best.pt`；另按 metallic val SRCC 存 `best_metallic.pt`（srcc_mean 峰值偏早、偏冻结，会**低估** metallic 专项改动，故单独追踪 metallic 峰值）。

### 2b.6 合成漏标负样本（synthneg，§5.5 决定性实验）
取**确实有金属**的资产（metallic 非黑像素 >20% 且原分 ≥3）→ 将其 metallic 贴图**清零** → 标 metallic=0（漏标）。仅加进 train，与原样本构成**最小对照对**（同物体，真图=高分 vs 黑图=0 分）。`zero_render_clip` 开关可同时清零 CLIP render 半段，作"去 render 循环捷径"的严格对照。

## 4. 逐条分析：每个改动为什么有效 / 无效

### 4.1 主干 ConvNeXt→DINOv2：小而稳的真实增益 ✅
全通道持平或微涨（~+0.01–0.02）。DINOv2 dense 自监督特征对材质纹理更友好。符合"已近天花板，强主干只能蹭一点"。

### 4.2 分辨率 224→448：几乎无红利，甚至有害 ❌
源图 2048²，224 丢 98.8% 像素，但 SRCC 几乎不动（normal 在 DINOv2 上反而掉）。**原因**：质量信号是低频/全局的（"一片死灰""异常色调""该不该是金属"都是粗判断），且**人类当初也是在正常观看分辨率下打分的**，超出人眼用到的细节对"预测人类打分"是无关信息。**教训：不是所有"信息量"都转化为"偏好相关信号"。**

### 4.3 文字提示 / 多输入辅助通道：无效 ❌
几乎零增益。pooled 辅助向量信息太稀，辅助通道与目标的关系没被显式正确建模。

### 4.4 多任务共享主干：最有效的单一改动 ✅✅
metallic 0.565→0.605（+0.04），全通道不掉。**机制**：训练时梯度共享，强通道逼主干学通用材质表示，弱通道白嫖。

### 4.5 EMD/NIMA 分布头：无帕累托代价的小赢 ✅
SRCC 持平回归（0.792）但 exact acc +0.02、asym acc 最高。序数头则 SRCC↑acc↓（帕累托权衡）。EMD 的"差多档重罚"对齐产品容忍度，选为生产头。

### 4.6 FiLM 条件化：无效，且暴露 bug ❌
给 metallic 头加 FiLM 每个 epoch 都更低。且发现 `_film` 实际取的是 base_color CLS 而非注释所说 render——即使用"好信号"条件化，FiLM 机制本身依然无效。

### 4.7 正则栈（drop_path + 辅助监督 + 强增广）：消 gap 但不涨 test ⚠️
见 §5.2——这是本项目最反直觉、也最有诊断价值的结果之一。

### 4.8 metallic 专项攻坚（全部失败/无效，逐条见 §5）
跨通道一致性架构、38k 重训、两阶段一致性分割、synthneg ×3——**四条独立路线，同一结论**。

## 5. 核心诊断（贯穿所有实验，本报告的核心贡献）

### 5.1 两种截然不同的失败模式
- **metallic = 结构病**：误差 U 形，各分数样本量均等。根因——**47% 的 metallic 贴图近乎全黑**（单图无信号）+ **标注自相矛盾**（"正确的全黑非金属"和"错误的全黑漏标"像素级完全相同、分数相反）。
- **roughness/normal/base_color = 稀有尾部病**：主体优秀，只在极少数极端分数崩（normal 满分仅 73 个）。这是数据量问题，靠采样/容量可缓解。

### 5.2 train-test gap 诊断：gap 是过拟合，但不是 metallic 的瓶颈 ⭐
用"train vs test SRCC 差距"区分"信息不够（连训练集都学不会）"还是"标注噪声/过拟合（train 高 test 低）"：

| 通道 | train SRCC | test SRCC | gap |
|---|:---:|:---:|:---:|
| roughness | 0.933 | 0.895 | 0.037 |
| base_color | 0.890 | 0.838 | 0.052 |
| normal_map | 0.868 | 0.808 | 0.060 |
| **metallic** | **0.788** | 0.627 | **0.161**（3–4×） |

metallic 的 gap 是其他通道的 3–4 倍，且 **train 能到 0.79** → 模型有能力拟合（信息够），但学到的是死记训练集噪声标签、泛化不了。

**但随后正则栈实验给出关键修正（§3.3）**：drop_path+辅助监督+强增广把 metallic gap 从 0.161 砍到 0.076，**而 test 纹丝不动（0.623 vs 0.624）**。这说明——
> **gap 是真的过拟合，但它不是限制 test 的因素。** 修复 gap 是让 **train 降下来贴近 test**，而非 **test 涨上去**。metallic test ~0.62 早被标注噪声卡死，过拟合只是"额外的死记"、不传递到 test。**test 不是 gap-limited，是 noise-limited。**

### 5.3 全解冻有害：39k 噪声数据撑不起微调 304M ViT-L
val srcc_mean 在 ep9（基本冻结、只解 stage4）见顶 0.806；ep15 全解冻 + LR 从 1e-4 重启后，ep16 断崖到 0.771，正则版甚至 ep20 自由落体到 0.52。**根因**：冻结时只有头部（几百万参数）在学，容量小够不着噪声→泛化好；全解冻 304M + 统一 1e-4（对预训练 ViT-L 高约 10 倍）→ 冲垮 DINOv2 预训练特征 + 过拟合矛盾标签。**修复**：LLRD 0.65 + 只浅解冻 stage4（gap 最小 0.067、不崩）。注意 best.pt 按 val 峰值存，部署权重始终是峰值（未受崩溃影响）。

### 5.4 metallic 是"看物体上下文"的判断，不是"看 metallic 图"的判断 ⭐⭐
这是推翻早期"信息缺失"悲观判断的关键认识。
- **近黑簇散布**：47% 的 metallic 图近乎全黑（输入近乎相同），但标签从 0 散到 5（<1%非黑 n=20585 的标签分布 [0分:3500,…,5分:7756]，std 1.89 > 整体）。同一张全黑图有人打 0（漏标）有人打 5（正确无金属）。
- **只看 metallic 图的天花板**：贝叶斯组均值 SRCC 仅 0.24–0.44。
- **但模型 test 有 0.62**——**已穿透图内天花板** → 模型在用 render/base_color 上下文判断"该不该是金属"，信息**部分就在输入里**，不是缺失。
- **干净的上下文信号在 base_color**（金属反照率，不随 metallic 图变；判别 AUC 0.93），**不在 render**（见 §5.6 循环论证）。

**按近黑/非黑子集拆开（emd_all，test，非黑=metallic 非黑像素占比）—— 一个重要的反直觉发现**：

| 非黑阈值 | 近黑 n | 近黑 SRCC | 近黑 MAE | 非黑 n | 非黑 SRCC | 非黑 MAE |
|---:|---:|:---:|---:|---:|:---:|---:|
| <2%（默认近黑） | 2289 | 0.555 | 1.11 | 2628 | **0.686** | 0.75 |
| <10% | 3417 | 0.575 | 0.98 | 1500 | 0.696 | 0.76 |
| <20% | 3988 | 0.588 | 0.95 | 929 | 0.700 | 0.74 |
| <30% | 4319 | 0.598 | 0.94 | 598 | **0.709** | 0.71 |

直觉会赌"非黑子集 ≈ 0.78–0.85（追平其他通道）、近黑 ≈ 信息地板"。**实测两头都不是**：
- **非黑（有金属）子集只到 0.68–0.71，越严阈值越高但 0.71 封顶——始终够不着其他通道（0.81–0.90）一档。** 说明 **metallic 难是全局性的：即使有金属可看、质量判断本身就比其他通道更吵**，瓶颈不只在近黑歧义。
- **近黑子集 0.55，高于 map-only 贝叶斯地板（0.24–0.44）约 0.15**——模型即使在全黑图上也用 base_color/render 上下文捞回一截，所以近黑**不是纯噪声地板**，而是"地板 + 上下文捞回"。

> 修正图景：metallic 不是"非黑很好 / 近黑见底"的干净二分，而是"非黑中上（~0.70）/ 近黑中下（~0.55）"。这**强化**了天花板结论——难度是全局的，单靠拆出近黑簇救不回整体。

### 5.5 决定性实验：合成漏标负样本证明天花板 ⭐⭐⭐（本报告最强证据）
**逻辑**：metallic 卡 0.62，是 (a) 信息真的不在输入里，还是 (b) 信息在、被噪声标签遮住了？训练侧已实证不动 test、天花板 proxy 又测不准（§5.7），唯一能区分的是"注入标准答案确定无误的样本看 test 动不动"。
- 做法：拿确实有金属的资产清零 metallic 图、标 0（人为制造无歧义的漏标），与原样本构成最小对照对（黑图=0 vs 真图=高分）。若模型能学会"base_color 显金属 + 黑图 = 差"，则 metallic 应涨、漏标召回应升。
- **结果（§3.4）：三个变体全部无效**。整体 SRCC 不破 0.62，**漏标检测召回反而下降**（0.556→0.338/0.415/0.504）。
- **所有 caveat 都已排除**：公平 best_metallic checkpoint（非早停）、加大剂量（5310）、去 render 捷径（norender）——结论一致。
- **机制**：黑图同时对应真 0 分（漏标）和真 5 分（正确无金属），模型用 base_color 上下文**区分不开这两者**，合成 0 样本只是给黑图叠加冲突监督 → EMD 分布变糊 → 黑图预测更不敢往低打 → 召回降。
> **结论：连手把手喂干净的漏标样本都用不上，说明区分"该是金属的黑图 vs 正确的黑图"的信息在单资产输入里确实太弱。这从正面证明了 metallic ~0.62 是噪声+信息天花板，不是模型能力问题。**

### 5.6 render 循环论证（解释了多次失败）
render 是用资产自己的（可能错误的）metallic 贴图渲出来的。漏标金属的坏件 render 也是哑光——**用 render 当判据预测金属，等于循环论证**，对最该抓的"漏标"bug 失明。解释了 FiLM(render)、SpatialAuxFusion(含 render) 全失败。synthneg-norender 对照进一步确认：去掉 render 后结论不变（仍不涨），说明 metallic 的瓶颈不是"被 render 捷径带偏"，而是 base_color 上下文本身就不足以消歧。

### 5.7 为什么无法直接测天花板
理想做法是测"测试-重测信度"（同输入重复标注的一致性），但数据是**单标注、无重复样本**。代理方法（近邻测试-重测 / 贝叶斯组均值）在像素空间和 frozen DINOv2 特征空间都试了，估计 0.24–0.55，**全部低于模型实际的 0.62**——逻辑上天花板必须 ≥ 模型，所以这些 proxy 系统性低估、作为上界无效（feature 不如微调模型能区分输入）。**这正是 §5.5 要靠"注入信息看反应"来旁证天花板的原因。**

## 6. 对照文献（这个问题是不是新的、该做到多少）

- **SRCC 该多少**：见 §1 标尺。roughness/base_color/normal 已在"真实场景/纹理 SOTA"水平，metallic 与最相近材质质量工作 BRDF-NQM（0.67）同量级。**不存在轻松上 0.9 的空间。**
- **是不是新问题**：**per-channel PBR 贴图质量评分（分通道）基本是空白**，无直接前人工作。相邻有 BRDF-NQM、DreamReward/Reward3D、3DGen-Bench 等——但都是**整体打分、不分通道**。这是可发表方向。
- **开源数据**：**没有**带"分通道 PBR 质量分"的公开数据集。原始材质库（MatSynth/AmbientCG/PolyHaven/Adobe）无质量标注；带质量标注的（3DGen-Bench 等）只针对生成 3D 整体。这份 49k 单标注数据在该细分方向可能是独家——既是护城河，也是无法对标的原因。

## 7. 结论与下一步

### 7.1 已确认的事实
1. roughness/base_color/normal 已逼近天花板，继续投入边际收益极低。
2. **metallic ~0.62 是硬天花板**（标注歧义 + 单资产信息不足），不是模型能力——四条独立路线（架构/训练侧/信息注入/反事实证伪）共同证明。
3. 唯一稳定有效的杠杆是 DINOv2 主干 + 多任务共享 + EMD 头，但增益小。
4. 全解冻有害（应 LLRD 浅解冻）；正则消 gap 不涨 test（noise-limited）；分辨率/prompts/render 注入/一致性分割/synthneg——均无效。

### 7.2 交付物
- **生产模型**：`dinov2_large_multitask_emd_all`（mean test SRCC 0.792），EMD 分布头取分兼顾排序与"5→4 可接受"的精度容忍。
- **推理/部署**：`pbr_infer/`（含 DEPLOYMENT_SOP）。
- **Demo 页**：`scripts/demo.py`（选资产看 4 通道贴图 + 预测分 vs 真分对比，含"metallic 差≥2 分"等筛选器，预计算预测不占 GPU）。
- **完整诊断脚本**：train-test gap、天花板估计（像素/DINOv2 空间）、synthneg 决定性 eval（近黑子群漏标召回）。

### 7.3 还想再提升 metallic 的两条路（都贵，需先证伪）
1. **多标注者去噪/重标**：对 metallic（尤其近黑歧义样本）做 2–3 人盲标，既能**直接测出人类一致性天花板**（确认 0.62 是否贴顶），又能用多数票/软标签去噪重训。这是唯一能确定"还有没有空间"的实验。
2. **金属语义先验（VLM），但先用低成本探针证伪**：不必搭完整 VLM 训练管线。先**离线跑一遍多模态 LLM**，对每个资产用 render/base_color 问"这个物体应该是金属吗"，得到 0–1"金属预期"标量（像预计算 CLIP 那样）。**第一步只做相关性检验**：该标量与"资产是否真有金属 / metallic 质量分"的 SRCC——若无相关（与 CLIP 文本探针 -0.072 一致），这条路也死；若有，再考虑当特征/门控注入。这把"VLM 大赌注"拆成几小时可证伪的小实验。

### 7.4 metallic 之外仍有空间的地方（已验证 ✅）
- **稀有尾部专项**（normal/roughness/base_color 的极端分数）：已把**尾部过采样**实现进多任务训练脚本（WeightedRandomSampler，按稀有极端分逆频率加权、跨 4 通道取 max）。**结果：normal 满分（test 仅 73 个）的召回从 10% → 61.6%，整体 normal SRCC 不降（0.807→0.805）。** 这是 metallic 之外真实有空间、且直接服务"筛高质量"产品目标的方向，**建议生产采纳**。详见 §8.3。

### 7.5 一句话收束
**这个项目的核心科学贡献不是"把分数刷上去"，而是用穷尽性实验 + 一个干净的反事实证伪，证明了 metallic 的性能上限由数据本身（标注歧义 + 单资产信息不足）设定，并精确定位了根因。** 在该约束下，当前模型（mean SRCC 0.792，三通道近天花板）已是合理的最优解。

## 8. 产品重定位与可交付能力（后期阶段核心）

前 7 节的北极星是"per-channel SRCC"。但 SOP 开头（baseline.md §1-5）写明项目真实目的是**自动分层筛选训练数据**。换到这个产品目标，结论从"metallic 卡死、悲观"翻转为"产品已健康"。

### 8.1 FinalScore 的真实构成（实测，R²=0.998）
从数据回归出真实加权公式（文档里的"RM 合并"已过时，实际 5 维独立打分）：
```
FinalScore = 0.189·baseColor + 0.351·normal + 0.152·roughness + 0.150·metallic + 0.147·rendered
```
| 维度 | 权重 | 与 finalScore 的 SRCC |
|---|:---:|:---:|
| **normal** | **0.351**（最高）| 0.831 |
| baseColor | 0.189 | 0.649 |
| roughness | 0.152 | 0.628 |
| **metallic** | **0.150**（最低）| 0.400 |
| rendered | 0.147 | 0.756 |
> **我们花最大力气死磕的 metallic，是产品里权重最低、贡献最小的通道；而权重最高（2.3×）的 normal 才是该投精力处——且它的瓶颈（尾部塌陷）可修。** 这是本阶段最重要的认知纠偏。

### 8.2 为什么"晚融合"是对的，不是妥协
模型是晚融合（4 通道独立编码 + 末端各自出分）。因为 FinalScore 本身就是各维度分的**加权和**（R²0.998）——**人类也是这么算的**，标签里**不含通道间交互项**。所以晚融合精确匹配标签生成过程；早融合/跨通道交互是在建模不存在的交互，这正是所有跨通道尝试（CrossChannelBlock / 破局点1 / 双流探针 / metal-prior 注入）一律无效的根本原因。

### 8.3 产品级指标（健康）
- **FinalScore 预测**：4 通道预测分 → finalScore，test SRCC **0.831**。
- **Tier 5 档分类**：exact 63%，**within-1 准确率 98.6%**（几乎从不错超过 1 档）。
- **高质量筛选**（"是不是 Tier1-2"）：二分 AUC **0.924**；保留 top5% 精度 0.66（6.4× 随机）、top30% 召回 0.92。
- **尾部过采样**（§7.4）：normal 满分召回 10%→62%，直接提升"识别高质量资产"能力。

### 8.4 缺陷检测能力（新交付物）
只做正例充足的 2 个缺陷（砍掉稀有的假AO 2%/法线反转 0.4%）。ConvNeXt-Base 共享主干 + 多尺度 AttentionPool2d + 路由头：

| 缺陷 | 主干 | AUC | AP | @召回80%精度 | @精度90%召回 |
|---|---|:---:|:---:|:---:|:---:|
| 有文字/印花 | ConvNeXt | 0.929 | 0.768 | 0.60 | 0.38 |
| 有文字/印花 | **DINOv3** | **0.948** | 0.815 | 0.70 | 0.41 |
| 法线异色 | ConvNeXt | 0.895 | 0.801 | 0.52 | 0.59 |
| 法线异色 | **DINOv3** | **0.931** | 0.855 | 0.68 | 0.69 |
> 可作审核工具：法线异色 90% 精度下自动 flag **69%**（DINOv3）。**DINOv3 升级**：其招牌 dense 特征对**局部缺陷**实打实提升（文字 +0.019、异色 +0.036 AUC）——这与"DINOv3 对 SRCC/metallic 天花板无用、但对 dense 任务有用"的分层判断一致。生产缺陷检测器用 DINOv3 版。`outputs/runs/{convnext,dinov3}_defect_text_tint*/`。

### 8.5 metallic 天花板：又加了三条独立证据（共 ~6 条）
本阶段在 §4.8 四条之外又补：
- **非黑/近黑拆分**：metallic 难是**全局性**的——非黑（有金属）子集也只到 SRCC 0.70，近黑 0.55；不是"非黑很好/近黑见底"的干净二分。
- **语义≠质量**：base_color 能判"是不是金属"（has-metal AUC 0.80），但这个先验对 metallic **质量分**相关近 0（-0.10）；显式注入该先验，集成增益 = 0。
- **roughness 双流（shuffle 对照）**：roughness 对 metallic 是冗余 + "好资产光环"，非新信息源；历史 F1=0.8447 是 F1（非 SRCC）+ 容量/光环的假涨。
- **ConvNeXt vs DINOv2 vs DINOv3（backbone 三重确认）**：90M ConvNeXt mean 0.783、308M DINOv2 0.792、**DINOv3 ViT-L 0.794**——三代主干在质量任务上**全部打平（±0.01）**，backbone 非瓶颈成铁案。DINOv3 的 metallic 单项最高（0.635）但近黑 0.570 仍在天花板带。DINOv3 的真实价值在**缺陷检测**（§8.4，dense 任务 +0.02~0.04），不在质量打分。
- **跨通道两种实现（metallic 读其他通道）**：
  - **cc_metallic（非对称池化 cross-channel，晚融合底座 + metallic 读 bc/nm/ro）**：metallic 整体 0.624→**0.632**，**近黑 0.555→0.570（+0.015，首个近黑非零正向）**，强通道完好（0.84/0.80/0.89）、无代价。微小但方向对，恰把近黑推向 ~0.58 的信息天花板。
  - **早融合（12 通道 input-stacking 联合编码）**：**净负**——mean 0.730、metallic 0.548、近黑 **0.480**（均低于 baseline）。根因是 12ch 对预训练 3ch backbone 是**分布外输入（OOD-stem 税）**，拖累全部通道，**不能干净检验**局部空间假设。
  - **局部空间跨通道（干净检验，避开 OOD 税）**——两种实现:
    - **me_spatialx（DINOv2，强通道独立 + metallic 的 patch token 空间 cross-attend bc/nm/ro 的 patch token）**：近黑 **0.557**、mean 0.792（强通道完好）。
    - **mid-fusion（ConvNeXt，多 stem→stage1 后 1×1 conv 合并→联合 trunk）**：近黑 **0.564**、mean 0.780。
> **跨通道总判读（池化 / 空间 / 全通道三种实现）**：近黑 SRCC 全部落在 **0.555–0.570** 带内（baseline 0.555 / 池化 cc_metallic 0.570 / 空间 me_spatialx 0.557 / 全通道 mid-fusion 0.564），**空间版甚至不及池化版**。即"局部空间跨通道"没有带来池化抓不到的信号——**在干净（无 OOD 税）条件下，"metallic 据其他通道判断"被证伪了**：区分"该是金属的黑图 vs 正确黑图"的信息确实不在其他通道里，无论池化还是空间地读。唯一附带结论：metallic-only 空间跨通道**无害**（me_spatialx mean=0.792，不伤强通道），是无代价但也无收益的设计。

### 8.6 文献定位（deep-research 验证）
- **没有**已发表工作直接给 metallic **通道质量**打分——这是真空白（可发表）。
- 工业界（Adobe Substance）用**规则化跨通道反射率校验**，且**明确声明无法判断意图/正确性**——印证我们"静态贴图判不出漏标"。
- metallic 缺失信息**只能从多视角/重光照渲染外观恢复**（IDArb：多视角 metallic MSE −57%；Hi3DEval 从多 HDRI 渲染感知判 metallicness）——但增益对 metallic 有界，且对"近黑漏标"仍受循环性限制。

## 9. 收尾结论

**项目定位升级**：从"per-channel 质量评分器"→**"资产质检（QA）系统"**。后者已经健康并可交付：
1. **质量分层**：FinalScore SRCC 0.831 / Tier within-1 98.6% / 高质量筛选 AUC 0.924 → 可直接用于"筛训练数据"。
2. **缺陷检测**：文字 0.93 / 法线异色 0.89，可作审核 flag。
3. **不确定路由**：EMD 头的分布熵可作置信度，把近黑等不可解样本转人工（架构已具备，未硬编码）。

**生产模型**：`dinov2_large_multitask_emd_all`（mean SRCC 0.792）+ 尾部过采样（救 normal 尾部）；EMD 取分兼顾排序与"5→4 可接受"。省成本可换 ConvNeXt-Base（−0.01 SRCC，−70% 参数）。

**诚实边界**：metallic ~0.62 是数据/标注设定的硬天花板（~6 条独立证据），**架构层已穷尽**。要再动它只有**资源决策**——多标注者重标（治标签噪声 + 测真天花板）或多视角重渲/VLM（补信息，增益有界），不是模型问题。

**一句话**：这个项目最终的价值不是"刷高了某个数"，而是 (a) 交付了一个产品级健康的资产 QA 系统（分层 + 缺陷 + 路由），(b) 用穷尽实验 + 反事实证伪精确划定了能力边界（metallic 天花板及其根因），(c) 纠正了北极星（normal/产品目标 > 孤立 metallic）。**知道哪里到顶、哪里还有空间、且把有空间处（normal 尾部、缺陷检测）真正做出来了——这本身就是结论。**

## 10. VLM 世界知识战役与 100k 扩充（2026-06-05 ~ 06-06）

§9 之后的三条战线一次性收口。判决性数字全部来自统一评估入口 `scripts/eval_multitask.py`（旧 test 严格同子集 n=4917，含近黑子集指标）。

### 10.1 数据侧：50k→100k（关闭）

数据集扩充到 104,621（旧 49k 切分逐行保留 0 错位，新增 55,471 按 pbrType 分层 8:1:1；datasets0604 软链接复用旧裁切，新增裁切全分辨率 2048²）。同配方（EMD-all + tailsamp）重训：

| 通道 | 50k | 100k | Δ |
|---|---|---|---|
| base_color | 0.8399 | 0.8450 | +0.005 |
| normal_map | 0.8059 | 0.8098 | +0.004 |
| roughness | 0.8931 | 0.9047 | +0.012 |
| **metallic** | 0.6233 | **0.6265** | **+0.003** |
| mean | 0.7906 | 0.7965 | +0.006 |

**双倍同分布噪声标签不平均掉噪声（第 10 证据）**。数据侧正式关闭。

意外发现：新批次 test 子集 metallic SRCC **0.695**（vs 旧 0.626），近黑占比相近（51% vs 46%）→ 非分布解释。旧批次近黑内 37% 打 2/3 分（全黑图打中间分=标注规则不一致）；新批次近黑 81% 打 0 或 5。**新批次标注实质性更干净，旧批次中间分是噪声藏身处。**

### 10.2 VLM 世界知识先验：探针有效，注入两审全败

- **Phase 1 探针**（Qwen2.5-VL-7B nf4，近黑 test 2289，输入=白模+albedo，render 因循环论证禁用，读 yes-token 概率）：v1"该含金属" 0.6275 / v2"全黑是错误" 0.6300 / v4"大面积金属" 0.6313 / **v2+v4 排名集成 0.6411** / 极端 0vs5 双峰 0.6766。**首次越过近黑视觉天花板 0.58**——世界知识真实有效但弱。bbox 定位路线死（0.54）。prompt 工程在 600 抽样上 +2.2 的增益全量缩水至 +0.3——小样本结论必须全量确认。
- **Phase 3 一审**（hidden 3584→128→zero-init 门控加进 metallic fused，49k 全量先验预计算）：metallic 0.6333 vs baseline 0.6233，但 **shuffle 对照 0.6337 同分同 best-epoch** → 增益不可归因（第 11 证据）。
- **Phase 3 二审**（regime 门控：近黑软路由 w=sigmoid((0.02-nb)/0.01)，先验梯度全部集中近黑，非黑物理关闭）：真先验 0.6172 vs shuffle 孪生 0.6217——**仍无可归因差异**（第 12 证据）。五个 run 的 metallic 散布 [0.617,0.634]、近黑 [0.543,0.561]，全在 run-to-run 噪声内。

### 10.3 人工看图裁决：标签语义错位是死结

对 VLM-标签分歧 top 案例人工看图：排球/蘑菇（VLM 判无金属✓，标签 0=扫描烘焙假材质）；钢楼梯/APEX 步枪（VLM 判该有金属✓，标签 5=风格化工作流允许烘焙）。**标签评"该工作流下这张图是否合格"，VLM 答"现实里该不该有金属"——升级 VLM 只会让它对标签的 AUC 更差**（钢楼梯更自信=离标签 5 更远）。pbrType 路由不解锁（physical 子集 AUC 0.636）。

同时确认 VLM 自身缺陷真实存在：钢楼梯只敢给 p_yes=0.095（7B+nf4+UV 碎片图+无思维链 → 该自信时不自信）。

### 10.4 结论与去向

- 对当前标签的 SRCC：特征侧（含外部世界知识注入）操作空间为零，是物理性而非工程性的；
- 可行去向：① 标签侧清洗（降权旧批次近黑中间分/二元化，新批次当锚）；② VLM 升级（Qwen3-VL 代差）用于**产品侧**漏标检出（以人工抽检精度为准，不以标签 AUC 为准）；③ 人工核验金标准近黑子集（~200 例）让评估对齐真相。
- 资产沉淀：49k 全量 VLM 先验（p_yes+hidden，对齐 cache 顺序）、统一评估入口 eval_multitask.py、100k 全套数据管线（cache/CLIP/CSV）。

## 11. VLM-as-scorer 全量化、两端重定义、全覆盖增益（2026-06-08 ~ 06-12）

§10 之后转向"换可信目标 + 把数据看全"。判决数字来自旧 test 同子集 n=4917（VLM 经 `scripts/vlm_scorer_eval.py`，存 preds+probs）。

### 11.1 标签噪声 A/B 挖掘 + B 组盲区：input-side 全死 ⭐⭐

三个独立架构（DINOv2 视觉头 / Qwen-VL 打分器 / ConvNeXt）在同一套噪声标签上训练，**一致偏离某条 GT** → 该 GT 是标注错误（无关归纳偏置不会同向同幅幻觉）。按**方向**分两类（疑似度>1.5，脚本 `mine_label_noise.py`，复核页 `/labelnoise`）：

| 通道 | A·模型高/GT低（真标错） | B·模型低/GT高（模型盲区） |
|---|---|---|
| metallic | 355（非近黑172=最硬重标目标） | 206 |
| normal_map | 161 | 31 |
| roughness | 63 | 19 |

**Oracle 上界分解**（随机剔点为对照）：metallic 0.65 天花板里，A 标签噪声拖 **+0.126**、B 模型盲区拖 **+0.060**，两者占可修空间 ~65%/31%。

**B 组（木椅/木桌：metallic 图正确全黑、GT=5，模型却打 ~2）是 reasoning gap 不是 input gap**——逐一证伪：① base_color（已显示是木头）② Claim Overlay ③ 全 4 通道一起送（multi-channel 10k SFT，B 组 2.13→2.11 纹丝不动）④ 零样本 Qwen3-VL-8B + 显式推理 prompt（B vs 对照分离度 **−0.04**，世界知识没启动）。模型看得到是木头，但学不会"木头→不该有金属→空 map=满分"。仅剩：更大判官 / 显式物体类别监督 / 接受 4% 局限。

### 11.2 两端重定义 + 守护分 + 温度读出 ⭐⭐

**生产只在乎 0 分（拒）和 5 分（精选），中间"勉强可用"不决策敏感。SRCC 被中间一大坨主导=错的头条**：metallic 全量 SRCC 卡 0.65，但只看两端 GT∈{0,5} 的 AUC 高达 **0.90**——模型分辨"明确垃圾 vs 明确精品"其实很强。

定义**守护分**（K=20% 分诊：真5精选率 0.4 + 真5不误杀 0.4 + 抓5AUC 0.2）。"埋没精品更贵"取舍下：

| 模型/读出 | 真5精选 | 守护分 | pred_std |
|---|---|---|---|
| baseline 期望读出 | 75.7% | 0.877 | 1.08 |
| σ=0.25 锐化软标签 | — | 0.830 ❌ | 1.17 |
| **极端加权(GT∈{0,5}×2.5) + T=0.25 读出** | **82.5%** | **0.900** 🏆 | 1.68 |

**温度读出**（期望 Σp·k → 锐化 p^(1/T) 再求期望，T=0.25）零重训撬两端；**极端加权训练**是撞 SRCC 天花板一长串负结论后**第一个真正奏效的训练侧干预**——前提是把目标从 SRCC 换成"守护精品"。动态范围塌缩（EMD 软标签把预测拉向均值，std 1.08 vs GT 1.71）从小毛病升级为硬伤。

### 11.3 全覆盖增益：纠正"数据侧封顶" ⭐⭐⭐

§10.1 的"数据侧关闭"**对 VLM-SFT 不成立**——混淆了两条轴：50k→100k 是 DINOv2 在**已全覆盖**上加资产（真递减）；而 Qwen full60k 实际只覆盖 60k/157k=**38%（asset×channel）对、每个看一次=欠覆盖**，根本没到平台。

双卡 DDP（手动梯度 all-reduce，`torchrun`，NCCL timeout 3h 防 val 期 barrier 超时）训满 **157k items=0526 train 全覆盖 1 epoch**（vanilla σ0.75，4 通道），test：

| 通道 | 60k(38%覆盖) | 160k(100%覆盖) | Δ |
|---|---|---|---|
| base_color | 0.832 | 0.851 | +0.019 |
| normal_map | 0.801 | 0.829 | +0.028 |
| roughness | 0.902 | 0.913 | +0.011 |
| **metallic** | 0.631 | **0.661** | **+0.030** |
| **mean** | 0.792 | **0.8135** | +0.021 |
| 近黑 srcc | 0.559 | 0.598 | +0.039 |
| auc_missing | 0.768 | 0.789 | +0.021 |

**全通道一致 +0.02~0.03，metallic +0.030 是所有干预里单项最大 SRCC 增益**；val@终点 metallic 0.681 仍在涨（1 epoch 或许仍欠训）。**但守护分（两端）全覆盖 ≈ baseline（0.876 vs 0.877，std 仍 1.08）。两个杠杆正交**：全覆盖→聚合 SRCC，极端加权+T0.25→守护分。叠加（全覆盖+极端加权）可能两个指标通吃=明确的下一步。

### 11.4 数据分布核实

直读两份 manifest：**0526 metallic 是人为均衡**（原始计数 0/1/2/3/4/5 桶 ≈8000，桶 1/3/5 恰好 8000=按 metallic 分桶截 8000/桶）→ 在 0526 测的 metallic SRCC 都是人为均匀分布上的。**0604 是 0526 严格超集**（49165 全在其中、标签 0 改动 + 新增 55471）；新 55k 用**更粗的标注协议**（metallic 双峰、baseColor 73% 挤在 3 且无 5 分、normal 只剩 {0,2,3}）——两批是两套不通约的评分尺度，混训有风险。当前全覆盖模型=**0526 train 全量 1 epoch，未用 0604**。

### 11.5 结论与去向

- **聚合最强模型**：0526 全覆盖（mean 0.8135 / metallic 0.661），纠正了"数据封顶"误判；
- **守护精品场景**：极端加权 + T=0.25（守护分 0.900），通用 Qwen2.5 已胜 metallic 专家；
- **下一步**：① 全覆盖 + 极端加权叠加（聚合+守护双赢）；② 上 0604 更大数据（注意协议不通约）；③ A 组非近黑标签清洗（Oracle 证 +0.126，唯一破天花板杠杆，需第三方裁判解循环）；
- 资产沉淀：双卡 DDP 训练入口（vlm_scorer_sft.py，--multi-channel/--extreme-weight）、A/B 标签噪声挖掘+复核页、守护分/温度读出评估。
