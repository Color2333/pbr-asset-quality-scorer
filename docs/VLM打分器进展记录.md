## 1. 范式与框架（我们具体做了什么）

核心思路：**不把 VLM 当特征提取器，而是让 VLM 直接当打分器**（Q-Align / DeQA 范式）。围绕这个思路,我们从零搭了一套训练+评估框架,逐项说明做了哪些事:

### 1.1 打分机制：从 VLM 里"读出"分数
- prompt 让模型输出一个 0–5 的数字;我们**不解析文本**,而是取 answer 位的 logits,只在 6 个数字 token（"0"…"5"）上做 softmax,得 6-bin 概率 `p`。
- **分数 = 期望值 Σ p(k)·k**（连续、保留序数性,等价于 DINOv2 的 EMD 头思路)。
- 评估时**额外存下整张 `probs.npy`(N×6)**,使得读出方式可以离线改——这让我们后来零成本做了**温度读出**(`p^(1/T)` 再求期望,T<1 锐化、对抗均值塌缩)和 argmax/期望对比,不用重训。

### 1.2 监督信号：DeQA 软标签
- 不用 one-hot,把整数 GT 用高斯软化成 6-bin 目标分布:`soft_target(score, σ)`,默认 **σ=0.75**。
- 损失 = answer 位上对软目标的交叉熵。**软标签是为了吸收标注噪声**("5 误标成 4"不会被当成完全错)。
- σ 做成可调超参,后来专门跑过 σ=0.25(锐化)对比(结论:锐化反而伤两端)。

### 1.3 微调方式：QLoRA
- 基座 4-bit NF4 量化冻结 + **LoRA**(r=16),`prepare_model_for_kbit_training` + 梯度检查点,单张 24G 卡放得下 7B。
- LoRA 目标:LLM 的 q/k/v/o/gate/up/down 投影;`--vision-lora` 可选把**视觉塔 blocks**(qkv/proj/mlp)也纳入(正则匹配)——因为法线/通道贴图是域外图像。
- 可训参数 ~47.6M(占 8.3B 的 **0.57%**)。试过 **DoRA**(权重分解 LoRA),因太慢(3.55s/项)放弃。

### 1.4 输入构造（make_msgs）
- 每个通道有专属评分准则 prompt(`CH_PROMPT`),明确"好的 X 通道长什么样"。
- 组图:`[待评通道图, 渲染图, base_color(非 base_color 通道时作上下文)]`——给模型"通道图 + 它在成品里的样子 + 材质底色"三重信息。
- **Claim Overlay**(metallic 专用):`make_claim_overlay()` 把 base_color 上"metallic 声称是金属"的区域**染红**,叠成第 4 张图 + 换专家 prompt——把"金属划分对不对"的跨图推理变成视觉模式匹配。
- 后来加了 `--multi-channel`:metallic 时把全 4 通道图一起送(验证"全上下文能否修 B 组盲区",结论:不能)。

### 1.5 采样：分层防淹没
- `stratified_items`:按 **(通道 × 分数) 配额**均匀抽样,保证稀有的极端分(0、5)不被中间分淹没。
- `--extreme-weight`:对 GT∈{0,5} 的样本 loss 加权,push 模型敢打极端(守护精品实验用)。

### 1.6 训练工程（让它跑得动、跑得对）
- **预取流水线**:`ThreadPoolExecutor`(DEPTH=6)在 GPU 算的同时,CPU 解码下一批 3× 2048² PNG(解码占单项耗时 ~40%)——把速度从拖不动优化到 ~0.94s/项。
- **quick-val + best-ckpt**:每 N 项在 val 子集上评 SRCC,按 val mean 存最佳权重。
- **双卡 DDP**(全量训练):没用 DDP 包装器(4-bit + 梯度检查点 + PEFT 组合易出隐性 bug),改**手动梯度 all-reduce**——分片数据、各 rank 同步梯度、仅 rank0 评估/存档、barrier 同步;并把 **NCCL 超时从 10min 调到 3h**(rank0 长 val 时 rank1 不被 watchdog 杀);`setsid` 启动脱离会话。
- **环境踩坑**:torch 降到 2.7/cu126(2.12 对驱动太新);权重走 **ModelScope** 下载(hf.co 被墙)。

### 1.7 框架里迭代试过的变体（全部留作开关）
σ(软标签宽度)、vision-LoRA(视觉塔适配)、max-pixels(分辨率)、Claim Overlay、multi-channel(全通道输入)、extreme-weight(极端加权)、metallic_stretch(输入增亮,DINOv2 侧)、温度读出。每个都是 config/CLI 开关,可复现对比。

**脚本**:训练 `scripts/vlm_scorer_sft.py`,评估 `scripts/vlm_scorer_eval.py`(存 preds + probs)。
**基座**:`Qwen2.5-VL-7B-Instruct`(主力)/ `Qwen3-VL-8B-Instruct`(对比)。

## 3. 关键里程碑

1. **范式验证成功**：零样本 0.027 → 10k SFT 0.729 → **全量 60k 0.792，1/16 数据暴露追平 DINOv2（0.792）**。证明 VLM-as-scorer 在 PBR 通道质量上成立。
2. **第二个独立强模型**：与 DINOv2 架构无关，使后续跨模型标签噪声挖掘、集成成为可能。
3. **集成**：DINOv2 + Qwen2.5 简单平均 metallic +0.024（0.655）；两模型相关性 0.811（最低）。

---

## 4. metallic 专家（Claim Overlay）与 B2 判决

- **Claim Overlay**：把 base_color 上"声称是金属"的区域染红 → 跨图推理变视觉模式匹配。
- 结果：专家 metallic **0.655**（>通用 0.63），**auc_missing 0.794**（>通用 0.75）。增益集中在 **missing-metal 检测**。
- **B2（决定性）**：在 B 组（206 个"非金属物体空 map 该高分"）上，专家把预测从 ~2.1 几乎没拉动（vs GT 4.4）。**Overlay 没修好 B 组盲区**——它验证"声称的金属对不对"，给不了"该不该有金属"的物体先验。
- **定位**：专家 = 优秀的 missing-metal 检测器，不是通用 metallic 打分器。

---

## 5. 两端 / 守护精品（重要方法论）

- **发现**：生产只在乎 0/5 两端；SRCC 被中间主导是错头条。**两端 AUC（0 vs 5）≈ 0.90**，远高于全量 SRCC 0.63。
- **守护精品指标**（埋没真5最贵）：真5精选率 + 真5存活 + 抓5AUC 合成。
- **温度读出（零重训免费胜利）**：期望读出 Σp·k 把预测压向均值（专家 std 1.08 vs GT 1.7）；换温度锐化 p^(1/T)，**T≈0.25 时守护分 0.831→0.865（专家old）/ 0.860→0.897（0604）**，真5精选 +7~10pct，pred_std 恢复到 1.5+。
- **极端加权训练 + T=0.25 = 新最佳守护分 0.900**（baseline 0.877，+0.023），真5精选 75.7%→82.5%。**这是第一个真正奏效的训练侧干预**——一旦目标换成正确的"守护精品"，训练就能帮上忙。
- **σ=0.25 锐化软标签：失败**（守护分反低于 baseline）。

## 6. 负结论汇总（已证伪，勿重做）

| 方向 | 结果 | 证据 |
|------|------|------|
| 数据翻倍 (50k→100k) | metallic **+0.003** | 数据侧关闭 |
| 换 backbone (Qwen3 全量) | metallic **−0.036**, mean 打平 | 不值得上全量 |
| 视觉编码器适配 (VLM vision-LoRA / DINOv2 解冻) | 落在同一 ~0.79 / 净有害 | 编码不是瓶颈 |
| 提高分辨率 (224→448) | ±0.01 噪声内，metallic 略降 | 分辨率非杠杆 |
| 多通道输入（攻 B 组） | B 组纹丝不动 (2.11 vs 2.13) | input-side 救不了 B 组 |
| 零样本世界知识判官（攻 B 组） | 分离度 −0.04 | 8B 判官也分不开 |
| metallic 先验注入 DINOv2 | 两审失败 | — |

**统一原因**：聚合 SRCC 被**单人标注的标签噪声**封顶；编码/容量/数据量旋钮全部撞顶。

## 7. 近黑 metallic 的关键反转（标签其实良定义）

- 我（视觉模型）盲标 40 个近黑金标：按"物体材质"判 vs GT **零相关（−0.09）**；非金属物体 GT 均匀铺 0–5，一度以为"近黑 GT 是噪声"。
- **拿到 SOP 后反转**：SOP 评的不是物体材质，而是 **metallic 贴图的"干净度/划分正确性"**（非金属区必须干净 0，大面积不合理灰=1分）。把 metallic ×8 增亮看真实杂灰、按 SOP 重判，**vs GT SRCC 从 −0.09 跳到 +0.50**。
- **机制 + 新杠杆**：判别信号是**极低幅度的杂灰**，原图近黑里几乎看不见，模型"看不清"。→ **给模型喂对比度拉伸的 metallic 图**（不动权重、只放大输入），让冻结编码器看见它。这是当前唯一有清晰机制、且没证伪的近黑改进方向。

## 9. 结论与方向

1. **VLM-as-scorer 范式成立**，1/16 数据追平 DINOv2；生产可用。
2. **天花板是单人标签**：编码/容量/数据/架构旋钮全试遍，皆封顶。
3. **生产选型（守护精品场景）**：Qwen2.5 通用版 + T=0.25 读出（守护分最高，且通用版打赢了 metallic 专家）。
4. **唯一活着的两条**：① 改输入让近黑信号可见（增亮，验证中）② 数据侧**多人重标**（破天花板的真杠杆，也是潜在论文核心）。
5. **专家定位**：metallic 专家 = missing-metal 检测器（auc 0.79），单独交付。
