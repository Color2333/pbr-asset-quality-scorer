Context：
When training any model, it is usually advised to use all "usable" data to train the model with as much prior as possible, and start shrinking the distribution with higher and higher quality data. Therefore, an ideal system is to label PBR maps with multiple quality levels.
任务流程：
目标
把已保证“三通道完整（BaseColor / Normal / MetallicRoughness）”的资产，自动化预检 → 人工打分 → 自动分层 → 导出训练集用到不同阶段

等级定义：
Usable：
Tier 1：电影级（Cinematic）（5分）
  - 定义：规范的高质量 PBR 贴图集，细节极其丰富且物理/视觉上正确。
  - 分辨率：主贴图通常 >= 4096 px（4K）
  - 特征：各通道细节层次丰富、无明显压缩伪影、PBR 通道间物理一致（例如金属区无漫反射）、渲染效果接近电影级材质。适合近景拍摄或高端渲染。
Tier 2：高级（PS5 / AAA）（4分）
  - 定义：规范的高质量 PBR 贴图，细节与材质表现优良，适用于主机/AAA 项目。
  - 分辨率：推荐 >= 2048 px（2K），最低可接受 >= 1024 px（视 asset 用途）。
  - 特征：法线、粗糙度/金属区分明确，BaseColor 曝光与色彩控制良好，少量可接受艺术化处理但物理合理性保留。
Tier 3: 游戏级（Game Quality / Indie ~ PS4）（3分）
  - 定义：可用于游戏（第八代主机 / indie）的大部分场景，整体正确且视觉满意。
  - 分辨率：写实类 >= 1024 px；风格化模型通常 512–1024 px 且表现清晰。
  - 特征：大多数区域细节可辨，几乎无严重瑕疵或可修复的小瑕疵；适合中距离视角。
Tier 4: 手游级（2分）
  - 定义：适合移动端或远景使用，有正确的材质关系但细节稀少。
  - 分辨率：通常 512–1024 px（以纹理压缩与性能为考量）。
  - 特征：结构与材质基本正确、无明显模糊或严重失真，但细节不足，适合远距或低预算项目。
Tier 5：勉强可用（1分）
  - 定义：贴图“能用”但存在明显问题（模糊、瑕疵或局部错误），需人工修复或仅作非关键用途。
  - 分辨率：通常 >= 512 px，但细节丢失或明显压缩痕迹。
  - 特征：存在局部错位、拉伸、压缩伪影或曝光问题；金属/粗糙度逻辑可能有瑕疵但整体还有可用信息。
Incorrect （0）#自动化80%，人工20%；
  - 定义：不可作为训练或直接使用的样本。
  - 典型情况：占位图（纯色、checkerboard）、通道错误（Normal 全灰）、BaseColor/金属通道不可用或严重错误（例如金属区有漫反射），或贴图明显被破坏/损坏。

评分要点：
1. 视觉表现分类（分类标注）
- 物理性
模型制作基于现实，追求写实还原，强调客观真实性，个人的主观艺术加工较少。


- 风格化
常见于卡通或具有独特美术设计的资产。制作重心在于强调风格特征与美术视觉表现力，为了达成特定的美学效果，允许在 PBR 规范上进行主观偏离（例如不严格遵循能量守恒定律，或对金属粗糙度进行非物理的艺术化改动）。



- 其它
  既不追求物理真实，也不具备典型的艺术风格化特征。 这一类别专门用于归纳那些无法被前两者标准覆盖的资产，通常脱离了常规的审美或物理交互逻辑。


- 存疑：【存疑项不参与评分】
若发现模型缺少 Alpha 贴图导致纹理显示异常或其他无法进行判断的模型，请统一将其归类为‘存疑’项。



2. Base Color（基础色）
- 5分：
  - 视觉表现：
    - 色彩层次极其丰富：高频细节 + 大面积渐变过渡自然；
    - 完整体现材质特征（皮革、金属、石头、漆面都“像真的”）；
    - 暗部不过黑、亮部不过曝，整体曝光完美，符合模型整体材质表现；
    - 无不合理脏色（Green Shift / Magenta 等）；
    - 纹理密度一致且极度精细（无重复块，无锐度损失）；
  - 直观感受：
    - “A站或3A游戏级别作品，反照率控制非常干净，颜色关系完全正确，纹理层次美观到能用在高质量游戏大作里。”

- 4分：
  - 视觉表现：
    - 色彩准确、材质特征明确；
    - 细节丰富但稍逊 5 分级别的精度；
    - 局部有轻微锐度下降或色彩偏差；
    - 过渡自然，但可能在某些区域比较“平”；
  - 直观感受：
    - “很优秀的游戏贴图，整体专业，但还没有那种‘惊艳’，细节 90% 到位。”

- 3分：
  - 视觉表现：
    - 大部分区域能表达材质；
    - 色彩整体正确但缺少细节层次；
    - 纹理稍模糊或锐度不一致；
    - 局部可能有明显重复块、轻微拉伸；
    - 过渡略僵硬（颜色跳变）；
  - 直观感受：
    - “能用，但明显缺乏打磨。属于可用，但不精致的贴图。”

- 2分：
  - 视觉表现：
    - 颜色单薄、缺乏分层；
    - 很多区域基本是一片色；
    - 纹理缺乏高频细节；
    - 重复、模糊、错位、脏色明显；
    - 难以辨认真实材质属性；
  - 直观感受：
    - “新手美术制作的贴图，有颜色但没有材质感。算不上真正的写实美术。”

- 0-1分：
  - 视觉表现：
    - 单色、纯灰、完全占位；
    - 贴图存在明显瑕疵；
    - 色彩严重偏离现实（亮紫、荧光绿）；
    - 完全无法体现材质；
  - 直观感受：
    - “美术表现效果非常糟糕，贴图表达与材质关联性极低...”
1分：（有区分颜色）

0分：（无区分色）


3. Normal Map（法线）
- 5分：
  - 视觉表现：
    - 大形体（低频）+ 中频（材质结构）+ 高频（微裂纹、纤维）结构完备；
    - 几乎无噪点、无反转、无奇怪光斑；
    - 光照移动时反射非常自然，形体紧实；
    - 烘焙无重影、无投射错误、无梯度断层；
  - 直观感受：
    - “法线烘焙非常干净，形体清晰有力，细节饱满，3A 资产。”

- 4分：
  - 视觉表现：
    - 整体干净；
    - 高频略弱，但形体表现准确；
    - 只有局部小区域噪点/瑕疵（可接受）；
    - 光照反应良好；
  - 直观感受：
    - “烘焙不错，细节略微粗糙但完全能用在大型游戏中，少量压缩或噪点”
- 3分：
  - 视觉表现：
    - 细节稍弱，不够“抓光”；
    - 局部略糊；
    - 中频结构不足（形体软）；
    - 看到轻微错烘、接缝；
  - 直观感受：
    - “能用但有点软，不够‘硬朗’，需要靠其他贴图补偿”
- 2分：
  - 视觉表现：
    - 明显噪点、多次烘焙痕迹；
    - 频率分布不均（局部超锐、整体发软）；
    - 光照一转就闪；
    - 接缝明显断裂；
  - 直观感受：”烘焙质量堪忧，细节脏乱且形体破碎“
- 0-1分：
  - 视觉表现：
    - 法线全灰；
    - 方向错误（红绿反转）；
    - 反光乱跳；
    - 完全看不出材质结构；
  - 直观感受：“完全损坏的无效法线，光影逻辑彻底崩坏，无法建立任何体感。”

4. Roughness （粗糙度）
- 5分：
  - 视觉表现：
    - 值域利用合理（0–1 有层次）、不同材质区分明显、物理一致；
    - 粗糙度分布自然：磨损处亮、干净处暗、边缘过渡自然；
    - 层次丰富但不嘈杂，表面能看到“真实材质的油脂/磨损/氧化规律”；
  - 直观感受：
    - “模型表面质感强，能清楚分辨材质的种类与新旧程度，完全达到 AAA 级水准。”

- 4分：
  - 视觉表现：
    - 不同物理表面（如皮革与金属扣）的粗糙度数值设定符合常规认知，质感清晰；
    - 灰阶层次合理，能够较好地模拟出物体表面的触感；
    - 整体细节丰富度略显平淡，缺乏细腻的高频特征；
  - 直观感受：
    - “材质区分优秀，但在微观细节的深度上稍显单一，局部存在少量可接受的噪点或量化痕迹。”
- 3分：
  - 视觉表现：
    - 整体基本成立；
    - 灰阶数值分布过于均一，或者生产过于规律的变化效果，缺乏应有的起伏与随机变化；
    - 材质看起来不够真；
  - 直观感受：
    - “能区分基础材质但细节贫乏或小细节表达不准确，质感表现生硬，缺乏复杂变化。”

- 2分：
  - 视觉表现：
    - 整体一片光亮或雾蒙蒙；
    - 呈现出廉价的“塑料感”或“橡胶感”；
    - 质感几乎看不出；
  - 直观感受：
    - “大面积极度均一或利用不当，材质感模糊不清，影响视觉真实性”


- 0-1分:
  - 视觉表现：
    - 一片黑 / 一片白；
    - 完全错误材质类别；
  - 直观感受：
    - “单纯的填充了不相关的数值，完全丧失了对材质的表达能力几乎无分区或数值错误。”


5. Metallic（金属度）
- 5分：
  - 视觉表现：
    - 严谨的金属与非金属划分，边界清晰。非金属区域为 0，无中间灰色地带的模糊定义；
    - 对于生锈、积灰、涂漆或磨损区域，金属度贴图的数值切换符合物理逻辑；
  - 直观感受：
    - “展现出符合物理的真实金属特征，有明确的金属质感，呈现出正确的光泽效果。”


- 4分：
  - 视觉表现：
    - 金属与非金属的区域判定逻辑正确，符合物体基本的材质构成；
    - 光泽表现贴合现实；
    - 细节表现稍弱，微观渐变、涂层剥落的精细边缘或多种金属复合表面的处理上略显概括；
  - 直观感受：
    - “整体看金属效果表现稳定，但在部分细节上略有欠缺。”

- 3分：
  - 视觉表现：
    - 金属区域划分正确，但反射表现不够优秀，缺乏金属特有视觉强度；
    - 边缘处理过于简单；
    - 局部可见轻微的压缩伪影；
  - 直观感受：
    - “金属定义正确但质感廉价，处于‘仅达标’水平。”


- 2分：
  - 视觉表现：
    - 金属部分表达含糊不清，界限模糊；
    - 反射表现暗淡无力；
  - 直观感受：
    - “看不出金属质感，划分过于抽象。”

- 0-1分:
  - 材质的金属度表达完全违背 PBR 物理原则（如金属/非金属属性判定存在底层逻辑错误）；
  - 视觉表现：
    - 属性判定彻底错误（如金属与非金属区域完全对调）；
    - 出现大面积不合理、无意义的中间灰度值；
    - 无法支撑基本的金属材质定义；
  - 直观感受：
    - 金属度数值与材质表达完全‘脱节’，（例如金属为非 0/1 范围异常） 

1分：（金属区域划分有错误，或出现不合理大面积灰色/亮灰白色）


0分：（金属判定完全错误/非金属判定错误）


6. 人工美学评分
指资深美术人员对模型在最终成像上的主观质感评估。评分不局限于单一参数，而是综合考量材质逻辑的合理性、结构轮廓的扎实度以及光影细节的自然感，是否有任何破坏视觉沉浸感的瑕疵。
- 5分：
  - 视觉表现：
    - 在标准预览球或实际引擎（UE / Unity HDRP）中效果极佳，近景检视非常干净；
    - 色彩、反照率、粗糙度、法线等整体协调，材质逻辑完全成立；
    - 光照变化下表现稳定，不闪、不脏、不突兀；
    - 细节层级完整（大形体、中频结构、高频细节都有）；
    - 纹理锐度自然，没有人工锐化感或模糊感；
    - 模型形体结实、剪影优美，结构与贴图的细节完美契合，无结构性拉伸；
  - 直观感受：
    - “可以直接放进 AAA 大作或 CG 短片里的材质。质感真实、高级、舒服，没有破绽。”


- 4分：
  - 视觉表现：
    - 主观上整体相当逼真，材质区分明确；
    - 近景可见轻微瑕疵（色噪、过度锐化、小瑕疵点），但不影响整体感受；
    - 粗糙度、色彩、法线大体协调；
    - 光照下表现自然，无明显破坏观感的 artifact；
    - 模型结构坚实；
  - 直观感受：
    - 比较专业、只是缺少一点‘顶级感’或有轻微粗糙之处。”


- 3分：
  - 视觉表现：
    - 中景和远景看起来基本合理，材质感可辨识，模型基础形体完整；
    - 近景明显存在不足：
      - 色彩层级单薄；
      - 粗糙度分布比较平；
      - 法线偏软或细节不足；
      - 局部模糊或纹理密度不一致；
    - 整体能用，但不够高级；
  - 直观感受：
    - “能用，但比较普通。”


- 2分：
  - 视觉表现：
    - 远景可以看，但稍微靠近就暴露问题：
      - 颜色单一 / 反照率错误；
      - 粗糙度一片死平；
      - 法线拉伸、噪点或错误明显；
      - 几何体存在明显的拉伸或形体逻辑错误；
    - 材质逻辑不清晰：金属不像金属、皮革像塑料等；
    - 整体真实感不足，显得廉价或不自然；
  - 直观感受：
    - “材质不成立，看得出是低质资产。远景有时还能凑合。”

- 0-1分：
  - 视觉表现：
    - 明显不自然或渲染错误：
      - 粗糙度全白/全黑；
      - Base Color 颜色污染明显；
      - 模型结构严重破损，法线方向错误（翻转、发光）；
      - 出现严重闪烁；
    - 看起来明显“出戏”，无法正常用于游戏或渲染；
  - 直观感受：
    - “存在一眼看到的问题，不能直接用于任何项目。”

分数计算权重/映射
前置规则
1. 所有样本在发给艺术家前，必须通过自动 precheck（通道存在、非占位图、Normal 非全灰、RM 非单值、格式可读、分辨率元数据等）。
2. 艺术家只对以下 5 个主观维度 打分（0–5 分）：Overall / BaseColor / Normal / Roughness+Metallic / Rendered。
3. 每个维度 0–5 分，分数为整数或允许小数（建议保留到小数点后一位或两位）。
计算权重：
拿到打分结果更优先，权重计算分级映射不着急
- Overall Quality：0.15
- Base Color：  0.30
- Normal：0.16
- Roughness + Metallic：0.26
- Rendered Appearance：0.13
（总和 = 1.00）
权重计算方式是一个动态，可以按照每个通道的0-5分存储，尽可能多的记录每一个标注环节的产物
如何计算最终得分
1. 对每个资产，收集艺术家给出的 5 个维度分数（每项范围 0 到 5）。
2. 将每个维度的分数乘以对应权重，得到每项的加权贡献。
  1. 例如：Overall 分数 × 0.15；Normal 分数 × 0.30；以此类推。
3. 把五项加权贡献相加，得到 FinalScore（范围仍是 0 到 5）。
4. 对 FinalScore 做四舍五入到小数点后两位，以便统计与分层。

FinalScore = 0.15×Overall + 0.16×BaseColor + 0.30×Normal + 0.26×RoughnessMetallic + 0.13×Rendered
等级映射规则：
- Tier 1（电影级）：FinalScore ≥ 4.50
- Tier 2（高级 / PS5 / AAA）：4.50 > FinalScore ≥ 3.50
- Tier 3（游戏级 / PS4 / Indie）：3.50 > FinalScore ≥ 2.50
- Tier 4（手游级）：2.50 > FinalScore ≥ 1.50
- Tier 5（勉强可用）：1.50 > FinalScore ≥ 0.50
- Incorrect（不可用）：FinalScore < 0.50 或 precheck 未通过
评分示例：
假设某资产的人工评分为：Overall=5、Base=3、Normal=4、RM=2、Rendered=3。
计算（文字）：
- Overall = 0.15×5 = 0.75
- Base  = 0.16×3 = 0.48
- Normal  = 0.30×4 = 1.20
- RM = 0.26×2 = 0.52
- Rendered = 0.13×3 = 0.39
- FinalScore = 0.75 + 0.48 + 1.20 + 0.52 + 0.39 = 3.34（四舍五入到两位是 3.34）
映射：3.34 属于 Tier 3（游戏级）（因为 2.50 ≤ 3.34 < 3.50）。



标注数据准备-工程侧（precheck ）：

图片渲染布局：
左上：Rendered Preview（标准灯光下的 Lit 渲染） — 真实感直观比较重要，放大视角与近景表现。
右上：BaseColor（UV 展开或直接贴到模型的 Albedo） — sRGB 显示，检查颜色/曝光/缝隙/平铺。
左下：Normal（可视化 RGB）或 Normal-shaded preview — 显示法线细节（可用法线可视化 shader）。
右下：Roughness（灰度） + Metallic（灰度或单独小块） — 若 RM 是 packed（R=metal, G=roughness），则显示解包后的两个灰阶图或一张组合图。
